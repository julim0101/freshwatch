# 배포해서 링크 공유하기

빌드 결과물(`dist` 폴더)은 서버가 필요 없는 정적 파일이라 어디에 올려도 동작합니다.
목 데이터로 돌아가므로 백엔드 없이도 전체 시연이 가능합니다.

---

## 방법 1. Netlify Drop — 가장 빠름 (2분, 계정 불필요)

```bash
npm run build      # dist 폴더 생성
```

1. https://app.netlify.com/drop 접속
2. `dist` 폴더를 브라우저 창에 **드래그 앤 드롭**
3. 끝. `https://랜덤이름.netlify.app` 링크가 즉시 생성됩니다

Site settings → Change site name 에서 주소를 `freshwatch-17jo` 처럼 바꿀 수 있습니다.
수정 후 다시 올릴 땐 같은 페이지에 dist를 다시 드롭하면 됩니다.

---

## 방법 2. GitHub Pages — 기존 프로토타입과 같은 방식

이미 `julim0101.github.io` 를 쓰고 계시니 흐름이 익숙할 겁니다.

```bash
# 1) GitHub에 새 저장소 생성 (예: freshwatch)
git init
git add .
git commit -m "FreshWatch AI 프론트엔드"
git branch -M main
git remote add origin https://github.com/julim0101/freshwatch.git
git push -u origin main

# 2) 배포 (gh-pages 브랜치로 dist 업로드)
npm install
npm run deploy
```

3. GitHub 저장소 → Settings → Pages → Source를 **gh-pages 브랜치 / (root)** 로 지정
4. 1~2분 후 `https://julim0101.github.io/freshwatch/` 로 접속

> `vite.config.js` 에 `base: "./"` 가 설정돼 있어 하위 경로에서도 이미지·JS가 정상 로드됩니다.
> 이후 수정할 때는 `npm run deploy` 한 번이면 갱신됩니다.

---

## 방법 3. Vercel — 자동 배포 (푸시하면 자동 갱신)

1. https://vercel.com 에서 GitHub 계정으로 로그인
2. Add New → Project → 저장소 선택
3. Framework는 **Vite** 자동 인식 → Deploy 클릭
4. `https://freshwatch.vercel.app` 생성

이후 `git push` 할 때마다 자동으로 재배포됩니다. 팀원과 함께 개발한다면 이 방식이 편합니다.

---

## 방법 4. AWS S3 — 프로젝트 아키텍처와 일치

발표에서 "AWS에 올렸다"고 말하려면 이 방법입니다.

```bash
npm run build
```

1. S3 버킷 생성 (예: `freshwatch-17jo`) · 퍼블릭 액세스 차단 해제
2. 속성 → **정적 웹 사이트 호스팅** 활성화 · 인덱스 문서 `index.html`
3. `dist` 폴더 안의 파일 전부 업로드
4. 버킷 정책에 퍼블릭 읽기 권한 추가

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "PublicRead",
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::freshwatch-17jo/*"
  }]
}
```

5. 엔드포인트 주소로 접속 (`http://freshwatch-17jo.s3-website.ap-northeast-2.amazonaws.com`)

CLI를 쓰면 한 줄입니다.

```bash
aws s3 sync dist/ s3://freshwatch-17jo --delete
```

HTTPS가 필요하면 CloudFront 배포를 앞에 붙이면 됩니다.
