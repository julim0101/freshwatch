## 0. 한 줄 요약

> AI가 재고·유통기한·수요·날씨를 분석해 **순이익이 가장 커지는 할인율을 추천**하고,
> 담당자가 **승인**하면 ESL·POS에 즉시 반영되어 폐기 손실을 줄이는 서비스.

AI는 가격을 자동 결정하지 않습니다. 추천만 하고 최종 판단은 사람이 합니다.

## 1. 화면 구조

```
로그인 (매장·권한 선택)
 └ 콘솔
    ├ [점포]  폐기위험 대응   ← 메인 작업 화면
    │        재고 모니터링
    │        승인 이력
    │        성과 리포트
    └ [본사]  본사 대시보드

             효과 검증
             정책 시뮬레이터

    팝업: 상세(손익 시뮬레이션) · ESL 현황 · 현장 PDA · 할인 정책 설정 · 온보딩 투어
```

```
로그인 → 폐기위험 대응 → KPI 확인 → 추천 검토(필요 시 조정) → 선택 승인
       → ESL 자동 반영 → 마감 시간대 재점검 → 성과 확인
```

## 3. 화면별 기능

### 3.1 로그인

| 요소 | 설명 |
| --- | --- |
| 매장 선택 | 서울역·양평·잠실·청량리 4개점 |
| 권한 선택 | 신선팀 담당자 / 점장 / 본사 운영팀 — 선택에 따라 접근 가능한 메뉴가 달라짐 |
| 좌측 브랜드 패널 | 핵심 성과 3종(폐기율 4.9→3.1%, 조정시간 32→6분, 월 612만원) 노출 |

데모 계정이 미리 입력되어 있어 버튼만 누르면 진입합니다.

### 3.2 폐기위험 대응 (메인)

**① 실시간 컨텍스트 바**
- LIVE 탐지 표시
- 날씨 연동 — "서울 비 · 23°C · 방문객 −12%" (기상청 데이터 반영)
- 매장시간 (대한민국 표준시 실시간 동기화)
- 다음 의무휴업 D-day 표시
- 오늘 처리 실적 배지 (승인 시 자동 생성)

**② KPI 4종** — 승인할 때마다 실시간으로 변합니다

| KPI | 의미 |
| --- | --- |
| 승인 대기 | 미처리 추천 건수 (D-Day/D-1/D-2 내역) |
| 오늘 폐기 위험 | 조치하지 않으면 오늘 잃는 금액 |
| 예상 매출 | 추천 적용 시 회수되는 매출 |
| 예상 폐기손실 | 승인 후에도 남는 잔여 손실 |

**③ 손실 흐름 차트 (워터폴)**
폐기 위험 → 회수 → 잔여 손실의 흐름. KPI 숫자와 정확히 일치하며 승인할수록 막대가 줄어듭니다.

**④ 마감까지 소진 예측**
시간축 위에 "할인 미적용" vs "추천 적용" 두 곡선. 사이 면적이 추가 판매량이며,
현재 시각에 세로선이 표시됩니다. 늦어질수록 회수 물량이 줄어드는 것을 보여줍니다.

**⑤ AI 승인 대기열**
- 필터: 전체 / D-Day / D-1 / D-2 / 카테고리별
- 상품별: 재고·원가·미판매 시 손실액 표시
- 할인율 ±1%p 조정 (0~40% 정수 단위, 본사 상한 40% 초과 불가)
- **2단 결재**: 30% 이하는 담당자 승인으로 확정, 30% 초과는 점장 최종 승인 필요
- 조정 시 "추천 40%에서 조정" 경고 표시
- 체크박스 다중 선택 → 하단 액션바에서 일괄 승인
- ESL 미적용 품목은 "수기 라벨" 태그 (무게 측정 상품)

**⑥ 마감 모드** (마감 4시간 전 자동 전환)
- 마감까지 카운트다운 + 진행 바
- 1시간마다 D-Day 상품 추천 할인율 자동 +5%p 상향
- 대기열이 D-Day·손실액 순으로 자동 재정렬
- "D-Day 전량 상한 적용" 원클릭 버튼
- 시연용 가속(1초=1분) 및 미리 보기 지원

**⑦ AI가 추천하지 않은 상품**
접기/펼치기 영역. "잔여 3일·소진율 92%로 조치 불필요", "행사 중복 할인 방지 규칙" 등
AI가 판단을 내리지 않은 이유를 기록합니다.

### 3.3 상세 팝업 (손익 시뮬레이션)

대기열이나 재고 목록에서 상품을 열면 나타납니다.

| 구성 | 설명 |
| --- | --- |
| 할인율 슬라이더 | 0~40% 1%p 단위 조정 시 예상 판매·판매확률·손익이 실시간 재계산 |
| 할인율별 기대 손익 곡선 | 정점(최적점)과 현재 선택 위치를 점으로 표시. 30% 미만 구간이 평평한 것은 자체 EDA 결과 반영 |
| 수요 예측 차트 | 최근 실제 판매량(막대) + AI 예측(점선) |
| AI 추천 근거 | 판매확률·할인 후 마진·추천 할인율 |

**이 화면이 "왜 하필 40%인가"에 대한 답입니다.**

### 3.4 재고 모니터링

- 전체 신선 재고 목록 (임박 상품만이 아닌 전 품목)
- 검색 · 카테고리 필터
- 정렬: 위험도순 / 유통기한순 / 재고많은순 / 회전느린순
- 회전율 50% 미만은 주황색 강조
- 행 클릭 시 상세 팝업

### 3.5 승인 이력

- 가격 변경 전건 기록 (일시·담당자·AI추천율·승인율·조정폭·결과·매출)
- **카테고리별 조정 패턴 분석** — 담당자가 AI 추천 대비 얼마나 낮추/올리는지 평균
- "추천보다 낮춰서 폐기 발생 N건" — 조정의 결과를 데이터로 확인
- **CSV 다운로드** (엑셀 호환, BOM 포함)

감사 추적용이자 모델 재학습 데이터입니다.

### 3.6 성과 리포트

- 이번 달 폐기 절감액 / 폐기율(도입 전후) / 추천 승인률
- 월별 폐기 손실액 추이
- 추천 처리 비율 (그대로 승인 / 조정 후 승인 / 반려)
- **AI 모델 성능** — MAPE 14.7%(도입 전 22.1%), 적중률, 채택률, 드리프트 상태, 카테고리별 예측 오차
- **미승인 건의 실제 결과** — 반려 8건 중 6건 폐기, 21.4만원 손실

### 3.7 본사 대시보드 (본사 권한)

- 4개 점포 폐기율 랭킹 (오피스 상권 잠실점이 4.2%로 최하위)
- 점포별 절감액·채택률·미처리 건수
- 전사 누적 절감액 추이
- **확산 로드맵** — 파일럿 4개점 → 수도권 30 → 전국 110개점
- 전국 환산 시 연간 약 13.6억 규모 (점포당 84.5톤 · 147,000원/톤 기준)
- 행 클릭 시 해당 점포로 전환

### 3.8 ESG 리포트 (본사 권한)

- 폐기 감축량의 **탄소(tCO₂e) 환산**
- 소나무·승용차 주행·가구 배출량 환산 지표
- **카테고리별 무게 vs 탄소 비중** — 축산은 폐기 무게의 21%지만 탄소의 58%
- 고객사 감축목표 대비 진행률

계수 출처: Eriksson et al.(2015) 슈퍼마켓 식품폐기물 탄소발자국 연구

### 3.9 효과 검증 (본사 권한)

- 적용 2개점 vs 대조 2개점 12주 비교 실험
- 폐기율 / 마진율 / 판매전환율 3종 비교
- 주차별 추이 곡선
- 통계적 유의성 (p=0.003, 95% 신뢰구간, 표본 수)
- 선행연구 대비 (Sanders 2024: 폐기 −20.8%, 이익 +2.9%)

### 3.10 정책 시뮬레이터 (본사 권한)

슬라이더로 정책을 바꾸면 전사 영향이 즉시 계산됩니다.

| 변수 | 범위 |
| --- | --- |
| 할인율 상한 | 20~70% (현행 40%) |
| 할인 시작 시점 | 유통기한 1~4일 전 |
| 자동 승인 임계값 | 0~30% |
| 적용 점포 수 | 4~110개점 |

결과: 전사 순효과(억원) · 폐기 감축률 · 점포당 절감액/마진 훼손 · 탄소 감축량
상한 50% 초과 시 현장 인터뷰 기반 리스크 경고가 표시됩니다.

### 3.11 ESL 현황 팝업

헤더의 "ESL N건 조치 필요" 배지를 눌러 엽니다.

- 오늘 전송 / 반영 완료 / 조치 필요 건수
- 전송 로그 (라벨 ID, 소요 시간)
- 실패 건 재전송, ESL 미적용 품목 라벨 인쇄
- 승인한 건이 로그 상단에 실시간 추가

### 3.12 현장 PDA 팝업

헤더의 휴대폰 아이콘으로 엽니다. 매대에서 사용하는 모바일 화면입니다.

- 바코드 스캔 시뮬레이션
- 큰 글씨로 추천가 표시, ±5% 조정
- 승인 시 데스크톱 KPI에 즉시 반영
- **오프라인 모드** — 통신이 끊겨도 승인이 기기에 저장되고, 복구 후 일괄 동기화

### 3.13 할인 정책 설정 (점장 이상)

- 최대 할인율 (본사 지침 상한)
- D-2 / D-1 / D-Day 기본 할인율
- 마감 할인 시작 시각
- 자동 승인 임계값
- 알림 설정

**AI가 임의로 움직이지 않고 본사 지침 안에서만 추천한다는 것을 보여주는 화면입니다.**

## 7. 시연 시나리오 (3분)

1. **로그인** — 매장·권한 선택 후 진입 (온보딩 투어 3단계 자동 표시)
2. **폐기위험 대응** — KPI 4종과 손실 흐름 설명. "오늘 32만원을 잃을 상황"
3. **상세 팝업** — 손익 곡선으로 "왜 40%인지" 설명. 슬라이더를 움직여 최적점 확인
4. **일괄 승인** — KPI가 실시간으로 변하는 것 확인. ESL 배지 변화
5. **현장 PDA** — 매대에서의 사용 흐름, 오프라인 모드
6. **본사 전환** — ESG 리포트(탄소)와 효과 검증(A/B)으로 확장성·증명 설명
7. **정책 시뮬레이터** — "상한을 올리면?" 질문에 즉석 시연

## 8. 한계 (정직하게 밝히는 부분)

- 표시된 수치는 공개 데이터(M5·Favorita·Dunnhumby)와 합성 데이터 기반 예시값입니다.
- ESL 하드웨어 연동은 표준 인터페이스만 구현하고 목 서버로 검증했습니다.
- A/B 실험 결과는 시뮬레이션 값이며, 실제 도입 시 재검증이 필요합니다.
- 축소 모형(고객 2,000명·상품 40종·4개 점포)이라 절대 수치보다 비율·패턴이 유효합니다.
